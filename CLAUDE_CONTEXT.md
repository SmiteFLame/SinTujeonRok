## 작업 방식 규칙

1. **이슈 기반 워크플로:** Claude가 코드/문서 작성 → 유저 검토 → 승인 → 커밋 → 이슈 닫기 → 다음
2. **파일 읽기/쓰기:** 프로젝트 폴더 내 자유롭게 (허락 불필요)
3. **허락 필요한 것:** 패키지 설치, `git push`, 이슈 close
4. **파일 명명:** 문서는 `##_이름.md` (이슈번호 접두어), 코드는 Unity C# 컨벤션
5. **용어:** 게임 내 용어는 반드시 `Terminology.md` 기준 한자어 사용 (카드→패, 덱→패진, 전투→패결 등)
6. **DevLog.md:** 작업 완료 시 개발 로그 업데이트
7. **커밋 메시지:** 이모지 + 한글 설명 + 이슈 번호 (예: `🏗️ 프로젝트 구조 설계 (#9)`)

## 핵심 게임 시스템 요약 (코드 작성 시 참고)

- **패 64장:** 8지역 × (6일반 + 2신물), 24품계(品階) 등급
- **패결:** 턴제(수/手), 데미지 = `(공×술위력배율) × (1-방어율) × 랜덤(0.9~1.1)`, 방어율 = `수/(수+500)`
- **팔목 상성:** 순환 가위바위보 (유리 ×1.25 / 불리 ×0.80)
- **술 4등급:** 약술/중술/강술/극술, **기운 6종** 트리거, **비술** 100게이지
- **관문 96개:** 8장×12관문, 난이도 ×1.0→×1,000 (지수적)
- **방치 보상:** `분당기본량 × 시간(분) × 관문보정 × 공력보정` (최대 12h)
- **아트:** "조선 수묵화 × 모던 캐주얼", 색상 #F5E6C8(한지)/  #2C2C2C(먹)/ #D4380D(주홍)
